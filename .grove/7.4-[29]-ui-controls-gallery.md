# 7.4-[29]-ui-controls-gallery

**Kind:** work

## Goal

The third ladder app (guide Step 7): a single-window **AppKit controls gallery** —
16+ distinct controls in a tidy two-column, sectioned layout — built against the
CL-family interface contract (ADR-0033) and **TestAnyware VM-verified**. Exercises a
broad slice of the generated AppKit surface: class **convenience constructors**
(`+buttonWithTitle:target:action:`, `+checkboxWithTitle:…`, `+radioButtonWithTitle:…`,
`+imageViewWithImage:`, `+[NSImage imageWithSystemSymbolName:…]`), bare `make-instance`
alloc/init for the rest, the inherited NSControl value setters
(`setDoubleValue:`/`setIntegerValue:`/`setStringValue:`), and a wide spread of **enum
constants** (button types, bezel styles, progress/level/date-picker styles, box types,
text alignment, image scaling).

Pure ObjC — no Swift-native residual — so like hello-window it loads `:load-residual nil`
and needs **no** `libAPIAnywareSbcl` dylib (VM provisioning = libzstd only).

## Context

This is the app hello-window's learnings flagged as the one that "forces" the
**exact-class init registry** decision (`aw-apply-init` does no superclass walk, so an
inherited typed init like `initWithFrame:` is invisible to `make-instance` on a subclass).
**Decision taken (recorded in learnings): the workaround, not the runtime fix.** Controls
are built either via their class convenience constructor or via bare `make-instance`
(alloc/init → `initWithFrame:NSZeroRect`) + `setFrame:` + setters — which is the idiomatic
modern AppKit construction path anyway (Apple steers away from manual `initWithFrame:`).
No control on the gallery needs a typed *inherited* init, so the superclass-walk runtime
change is not required here; it stays a clean future enhancement.

## Controls (16+)

- **Buttons & toggles:** push button, checkbox, radio pair, NSSwitch, NSSegmentedControl.
- **Value selectors:** NSSlider (ticks), NSStepper, NSLevelIndicator (rating),
  NSProgressIndicator (determinate bar) + NSProgressIndicator (spinner, animating).
- **Pickers & fields:** NSPopUpButton, NSComboBox, editable NSTextField (placeholder),
  NSSecureTextField, NSColorWell, NSDatePicker.
- **Display:** NSImageView with a tinted SF Symbol.

## Done when

- `ui-controls-gallery.lisp` + `run.lisp`/`dump.lisp`/`build.sh`/`README.md` build a
  standalone `UIControlsGallery.app` (pre-flight + revive smokes green).
- TestAnyware VM-verifies the window renders all controls correctly; report + screenshot
  under `test-results/ui-controls-gallery/`; `learnings.md` records the init-registry
  decision and any findings.

## Notes

Layout is manual frames (proven in hello-window/probe; bottom-left origin, a top-down
cursor helper), grouped under bold section headers with NSBox separators. Window is
fixed-size (titled | closable | miniaturizable) — no Auto Layout, so controls do not
reflow on resize.
