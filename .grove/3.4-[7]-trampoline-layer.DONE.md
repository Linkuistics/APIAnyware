# 3.4-[7]-trampoline-layer

**Kind:** planning

## Goal

Design the contract's **lower layer** in detail — `libAPIAnywareSbcl`, the
trampoline-only Swift dylib that is the sbcl target's slice of the complete-API
model — and **assemble the SBCL target design spec** as the synthesis of leaves
010–040. Raise the sbcl-trampoline ADR.

## Context

Read: CONTEXT.md *`libAPIAnywareSbcl` / sbcl trampoline layer* + *Complete-API
binding model* / *Trampoline* / *Trampoline elision* / *`objc_exposed`*; the model
ADRs **0025** (model + elision) and **0026** (`objc_exposed` IR fact + emitter
contract); the **direct precedent ADR-0029** (gerbil grows a trampoline-only Swift
dylib — SBCL is in the identical position: cannot compile Swift inline); ADR-0028
(chez trampoline + shared-source call) and the racket trampoline spec
`docs/specs/2026-06-15-racket-trampoline.md` (the marshalling taxonomy + deferred
buckets + known-good exemplars — same shared IR → identical residual). The IR
boundary spec `docs/specs/2026-06-15-ir-objc-exposure-boundary.md`.

## Decisions to carry in (settled upstream)

- **Per-target hermetic, trampoline-only** (Q7 fork closed by ADR-0011/0029): no
  family-shared substrate; the dylib does not absorb the MOP runtime.
- **ObjC is direct** (`objc_msgSend` via `sb-alien`, trampoline elided); only
  `objc_exposed == false` residual (Swift-native funcs/constants + method frontier)
  + pointer-constants route through the dylib's flat C-ABI re-exports.

## What to design here

- The `libAPIAnywareSbcl` SwiftPM target + the generated `Trampolines.swift`
  emission pass (model on `run_gerbil_trampolines`/`run_chez_trampolines`);
  content-addressed `@_cdecl` entry naming so the sbcl emitter reconstructs symbols
  with no shared counter (ADR-0013 precedent).
- **`sb-alien` binding** of each C-ABI entry: one typed `define-alien-routine` /
  `alien-funcall` per trampoline signature (compiled-FFI, ADR-0015 — marshalling
  Lisp-side, like chez/gerbil; no native marshalling hop).
- **`save-lisp-and-die` interaction:** how the dumped image re-links/loads the
  dylib and how it composes with the **startup re-resolution pass** designed in
  `020` (does the dylib's own load-time setup absorb part of the `Class`/`SEL`
  relive burden, per 020 §B5?). Self-containment for `070` (`bundle-sbcl`): vendor
  + relocate the dylib (gerbil ADR-0029 §3 relocation precedent).
- **Residual scope:** expected to reproduce racket/chez/gerbil's residual exactly
  (deterministic from shared IR — gerbil hit 51 funcs + 7 constants). Note the
  deferred buckets (generics, async, non-bridged-struct/closure/unnameable params).
