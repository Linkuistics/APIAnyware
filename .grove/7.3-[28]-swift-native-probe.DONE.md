## Outcome (COMPLETE 2026-06-21)

✅ **Built + VM-verified.** Window "Swift-Native API Coverage" (640×300, centred) renders all
four trampoline shapes with live values from real Swift-native symbols, each reached only via
`libAPIAnywareSbcl`'s `aw_sbcl_swift_*` `@_cdecl` trampolines (bound from the GENERATED
bindings):

1. **function** — `CoreGraphics.hypot(3,4)` → `5.0`
2. **constant** — `Foundation.NSNotFound` → `9223372036854775807`
3. **class-owner init** — `NSNumber(integerLiteral: 42)` → a real `ns:ns-number`, `intValue` 42
4. **class-owner method** — `Scanner("APIAnyware:SBCL").scanUpToString(":")` → `"APIAnyware"`
5. **value-opaque box** — `IndexSet(5)`→`insert(7)` opaque-handle round-trip (mirrors the 050
   smoke's D5; the value-STRUCT-owner CLOS modelling stays the parked 090 leaf)

TestAnyware confirmed the window/title/size/app-identity semantically, the five live values
visually (OCR), and Cmd-Q terminated cleanly. VM provisioning: **two** dylibs — `libzstd`
(hello-window's finding) + the NEW `/tmp/libAPIAnywareSbcl.dylib` (the §6d residual), which
the dumped image auto-reopens at revive (ADR-0038 §5 — the FIRST app to dump+revive with the
dylib; the residual survives `save-lisp-and-die`). Artifacts:
`apps/swift-native-probe/{swift-native-probe.lisp,run.lisp,dump.lisp,build.sh,README.md,
learnings.md}` + `test-results/swift-native-probe/{report.md,swift-native-probe.png}`.

**Forcing-function findings (first app to load a `functions.lisp`):**

- **BLOCKER FIXED (emitter):** `coregraphics/functions.lisp` named nine `CGAffineTransform*`
  formals `t` (the C param), a CL defined constant → illegal lambda-list variable → the file
  would not load. Fixed in emit-sbcl (`naming::is_cl_reserved_formal`, applied in both
  `arg_name`s → fall back to `argN`); regenerated; goldens unchanged + new test. Protects
  every residual-using app.
- **FINDING captured to the grove inbox (cross-cutting, NOT fixed here):** Swift-overlay class
  names (`Scanner` for the ObjC `NSScanner`) are baked into `register-objc-class`, so auto-wrap
  + `make-instance` can't reach `ns:scanner` (only the pointer-based method trampoline works).
  Worked around via `(make-instance 'ns:scanner :ptr id)` over the real `"NSScanner"`. IR-level,
  likely shared across targets; does NOT affect the AppKit GUI ladder. Recommend a dedicated leaf.
