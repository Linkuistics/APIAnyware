## What this records

The **AppSpec toolkit grove** (`~/Development/AppSpec`, branch `appspec-toolkit`) is running to completion
during the `appspec-grove-pause-k66` pause. Its `forward-gen-suite-k24` leaf (under `acceptance-test-k21`)
**forward-generates** hello-window's `#lang app-spec` scenario suite from the firmed spec's §10 and authors
the per-app run-values config — committing them **into this worktree** (ADR-0013: AppSpec drives the run +
holds no app data; the data's home is APIAnyware). This leaf records that landing: the scenario suite +
run-values config that the live VM run (`04`/`05` upstream) replays against the four built impls.

## Authored in this commit (the forward-gen suite + run-values landing)

- **`apps/macos/hello-window/scenarios/`** — the forward-generated `#lang app-spec` scenario suite, three
  files, one scenario each:
  - **`01-steady-state-observations.rkt`** — the pure-observation cluster (one VM launch): process-running,
    the `Hello Window opened.` launch-log, the `Hello, macOS!` greeting + stable `Hello from` title
    substring (OCR), the window / static-text-label / Quit-menu-item AX nodes, and the discriminating
    `(expect-no-ax #:role 'AXTextField)` static-label negative.
  - **`02-command-q-terminates.rkt`** — the mandated §6 Command-Q→terminate invariant, isolated, asserting
    the negative `(expect-running-app … #:running? #f)`.
  - **`03-close-button-keeps-running.rkt`** — the `(to confirm in-VM)` close-button line as a provisional
    **`recording:`** scenario (gui-app keeps running per §3.8; a PASS confirms, a FAILURE is a
    spec-quality finding — ADR-0010 D4).
  - Every verb call carries a `;; spec:` back-trace; run-supplied values flow through `(run-value 'key)`.
- **`apps/macos/hello-window/run-values.rkt`** — the per-app `#lang app-spec/run-values` config carrying
  the window close-control coordinates (`close-button-x`/`close-button-y`, framebuffer px). Deliberately
  **NOT** under `scenarios/` (the runner sweeps every `.rkt` there as a scenario); consumed by
  `runner/main.rkt --run-values <this file>`. Per-impl `bundle-id` stays in each impl descriptor (k67);
  the runner merges the two into the single `current-run-values` table.

## Validation (AppSpec-side, recorded in the toolkit)

The suite was forward-generated by **two independent `forward-gen` subagents** (consensus — they agreed on
every §10→verb mapping and every cluster), reconciled to one suite, and graded against the **forward-gen
Tier-2 bar** (`capabilities/forward-gen/validation.md` §6): **L1a–L1d structural PASS** (all load; full
§10 coverage-or-gap; clean clustering; the recording tag) and the **anchored rubric (F1,F2,F3) =
(Pass,Pass,Pass) = accepted** across independent identity-blind grader runs — **meets the §4 accepted
bar**. The Tier-2 generation grade is recorded AppSpec-side in `forward-gen/validation.md` §6 (the
validation record is toolkit residue; the suite is app data → downstream, ADR-0013/0052).

## Findings carried back (for APIAnyware on resume)

- **Four §10 lines stay deferred gaps, not assertions** (the bar is bounded by the closed vocabulary):
  window **size** (§10.6) and **position** (§10.7) need `expect-ax #:size`/`#:position` (gap 2); the Quit
  item's **Command-Q key-equivalent** (§10.8) needs an AX key attribute (gap 2); the dynamic **"no
  interactive editing"** (§10.10) needs an exact/negative OCR verb (gap 3, covered *structurally* by the
  `expect-no-ax #:role 'AXTextField` negative). Turning a still-open gap into a hard assertion before its
  verb exists is the forward-gen mutant-D failure — so the suite **reports** them, never forces them.
- **The close-button coordinates are PROVISIONAL pending the live VM.** `run-values.rkt` hardcodes
  `(332, 284)` derived for an assumed 1024×768 non-HiDPI framebuffer (cirruslabs macos-tahoe-vanilla);
  `04` measures the real framebuffer (`testanyware screen size`) + window origin and refines the two
  literals. By design: scenario `03` is a `recording:` scenario, so a miss is a `04`/`05` adjudication
  finding, not a generation defect (ADR-0011 hardcodes geometry, derivation-from-AX deferred).
- **Async settle is run-tuning for `04`.** §10 maps termination/hide with no settle; the suite mirrors
  that (every verb traces to a §10 phrase). If a single-shot `expect-running-app` races the run-loop
  unwind, `04` inserts the in-set `(wait seconds)` — a run-capability tweak, not a §10 assertion (flagged
  in the `02`/`03` comments).
