# 4.2-[10]-object-model

**Kind:** work

## Goal

Emit the ADR-0034 CLOS object model — the heart of the emitter. Three modules:

- **`class_graph.rs`** — reify the ObjC ancestor chain for a framework's classes
  into emission order + parent links (peer `emit-gerbil/class_graph.rs`). The full
  ancestor chain becomes `defclass`es rooted at the runtime-owned `ns:ns-object`
  (which carries the foreign `ptr`); emit superclasses-before-subclasses.
- **`emit_class.rs`** — per class emit `(defclass ns:<cls> (<superclass>) (<slots>)
  (:metaclass objc-class))`. Slots: opt-in **foreign ivar slots** with baked
  bit-offset + foreign ctype (ADR-0034 §4 — `:offset`-bearing slot specs the
  runtime's `direct-slot-definition-class` picks up); the always-safe accessor path
  is the default, baked offsets opt-in. Also bake the **Class/SEL string tables**
  the runtime startup re-resolution pass consumes (ADR-0034 §6 — class identity by
  string, never a baked pointer).
- **`emit_generics.rs`** — per distinct selector emit **one** `(defgeneric ns:<sel>
  (receiver …) (:documentation …))` in `ns:` (the named contract surface, §3.2),
  then per (class × selector) **one** `(defmethod ns:<sel> ((self ns:<cls>) …) …)`
  specialized on the receiver, body = the direct `objc_msgSend` `sb-alien` crossing
  (typed `alien-funcall` per ABI signature, ADR-0015/0038 §3). `objc_exposed ==
  false` methods get **no** direct body — collect them for the trampoline residual
  (050); the §6d `554 method` + `576 init` originate here. **No** `generics.ss`-style
  sharding, no special flags, no parallel-compile (ADR-0034 §3 — the explosion does
  not reproduce in native CLOS).

`make-instance`→alloc/init and subclass synthesis are **runtime** (050); this leaf
only bakes the graph + generics + slot specs.

## Context

ADR-0034 (whole ADR — metaclass, D6 dispatch, slots §4, make-instance §5, static
emit + re-resolution §6) is the spec. SBCL design spec §2. Contract §3.2/§3.3.
Reference: `emit-gerbil/src/{class_graph.rs,emit_class.rs,emit_generics.rs}` — but
gerbil emits a manifest defclass graph WITHOUT the MOP + a `:std/generic` veneer;
SBCL emits `:metaclass objc-class` + native `defgeneric`/`defmethod`. The
`objc_exposed`-driven direct-vs-residual method split is shared (ADR-0026 §3).

## Done when

- `emit_class`/`emit_generics` produce the defclass graph + per-selector generics +
  per-class methods for a fixture framework; snapshot tests pass.
- Foreign slot specs carry baked offset+ctype; baked Class/SEL string tables emitted.
- `objc_exposed == false` methods/inits collected for the residual, not emitted as
  direct `objc_msgSend` bodies.
