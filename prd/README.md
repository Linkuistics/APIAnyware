# prd/ — Product Requirement Documents

Human-facing, team-shareable agreement checkpoints, kept as a small, focused
top-level directory (the same cross-cutting-record carve-out ADR-0045 makes for
`adr/`): a PRD belongs to no single domain and is never retired. Relocated from
the former `docs/prd/` in `co-locate-docs-k9`.

Like `adr/`, this is *not* the "large top-level `docs/`" REFACTOR §10 forbids —
it is a single-purpose record dir for agreement points owned by no domain.
