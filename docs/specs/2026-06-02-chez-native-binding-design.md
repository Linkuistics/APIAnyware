# Chez native-binding design — the compiled `foreign-procedure` seam is already thin

**Status:** design (grove `chez-adopt-native-binding`, leaf `010-design-and-spike`);
backed by the measurement spike in `docs/research/2026-06-02-chez-dispatch-spike/`
(`FINDINGS.md` + repro harness). Decisions recorded as **ADR-0015** (dispatch +
marshalling stay in Scheme) and **ADR-0016** (outbound callbacks; foreign-thread
activation).

**Audience:** anyone extending the `chez` target, or comparing it to the `racket`
target's mirror decision. Assumes `CONTEXT.md` (Target, Target idiom,
`objc-object`, Entry-point autoreleasepool), **ADR-0005** (idiomatic Chez),
**ADR-0007** (chez lifetime model), **ADR-0010** (the per-target native library
*is* the binding), **ADR-0011** (targets hermetically isolated), and the racket
analogue `2026-05-31-racket-native-binding-design.md` (reused as *reasoning*, not
code — ADR-0011).

## Scope

How the `chez` target crosses into macOS — method dispatch, argument/result
marshalling, callbacks, memory, threading — and what adopting ADR-0010 + ADR-0011
*actually requires* for chez. The headline: chez is **already** largely
ADR-0010-compliant, for a reason that does not hold for racket.

---

## 1. The objective function, and why chez's answer differs from racket's

ADR-0010 makes the per-target native (Swift) library *the* binding: a fat native
core behind a thin, static FFI crossing. Racket took the strongest reading —
*relocate dispatch and marshalling into generated native entries so the scripting
side never assembles an `objc_msgSend`* — because its FFI crossing was
**interpreted/macro-heavy** (`tell` ~90 ns, `get-ffi-obj` msgSend ~10 ns), and a
native typed entry measurably beat it.

**Chez's baseline is different in kind.** A fully-typed `foreign-procedure`
compiles to a *direct* C call to `objc_msgSend` — there is no interpreted or
macro layer between Scheme and the C ABI. The spike (`FINDINGS.md` Part A)
confirms it empirically: chez's direct `foreign-procedure` is **already at the
native dispatch floor** (~6 ns simple shapes, ~10.5 ns struct return), and
wrapping `objc_msgSend` in a native shim is *equal* on simple shapes and **~3×
slower on struct returns**. Marshalling (Part B) tells the same story: chez's
crossings are so cheap (~6–17 ns) that moving string/collection coercion native
saves at most ~12% and usually *loses*.

> **Conclusion.** For chez, ADR-0010's "thin static crossing" is satisfied **by
> `foreign-procedure` itself**. The way to honour ADR-0010 is *not* to relocate
> Scheme work into the native library — doing so would add hops and **violate
> ADR-0010's performance goal**. It is to keep the cheap compiled crossing, keep
> dispatch/marshalling idiomatic in Scheme (ADR-0005), and reserve the native
> library for the concerns that genuinely cannot be expressed as a thin call:
> blocks, delegates, dynamic classes, GC pinning, and foreign-thread activation.

This is the same objective function as racket; the measured answer is the
opposite, because the baselines differ.

---

## 2. Dispatch — keep direct `foreign-procedure` (ADR-0015)

Measured ns/call, chez 10.4.1 / arm64, 3M iters, steady state (`FINDINGS.md` §A):

| ABI shape | direct `foreign-procedure` | native typed shim | libffi |
|---|--:|--:|--:|
| scalar | **~6–9** | ~6–8 | ~21 |
| id→id | **~17** | ~17–18 | — |
| 2×float | **~6.4** | ~6.8 | — |
| struct ret (NSRect) | **~10.5** | ~33 | — |

**Decision:** the emitter keeps open-coding one typed `foreign-procedure`
`objc_msgSend` per method signature at the call site (status quo). **No generated
native dispatch library; no emitter cutover.** This is the deliberate divergence
from racket's ADR-0013, on measured grounds. (Recorded: **ADR-0015**.)

The combinatorics that *cost* racket nothing (it generates per-signature entries
from the IR) cost chez nothing either — but chez doesn't need them: the typed
`foreign-procedure` *is* the per-signature entry, generated by the emitter into
Scheme and compiled by Chez. libffi-generic is retained conceptually only as the
escape hatch for any signature the emitter cannot type statically (variadics are
already filtered); the spike shows it is ~3× slower, so it is a fallback, not a
path.

## 3. Marshalling — stays in idiomatic Scheme

The marshalling-depth spectrum racket walks (Depth 0 opaque → Depth 1 typed →
Depth 2 batched) is a *response to expensive crossings*. Chez's crossings are
cheap, so the spectrum collapses: **Depth 0/1 in Scheme is already optimal.**
`FINDINGS.md` §B: native string round-trip saves ~12% (NSString work dominates,
not crossings); no native collection batch beats chez's direct per-element loop.

**Decision:** `types.sls` keeps its geometry ftypes and its
`string<->NSString` / `list<->NSArray` / `hash<->NSDictionary` converters in
Scheme. String conversion continues to use the native `aw_chez_str↔ns` helpers
(one cheap crossing each) — those are kept because they're native *primitives*
chez calls, not because marshalling was relocated. (Part of **ADR-0015**.)

## 4. Callbacks / delegates / blocks — outbound, with foreign-thread activation (ADR-0016)

The *inbound* path (ObjC → Scheme) is governed by a hard constraint the spike
nailed down (`FINDINGS.md` §C).

### 4.1 Findings
- A Chez `foreign-callable` entered from **any Scheme-unregistered OS thread
  crashes hard** ("nonrecoverable invalid memory reference") — confirmed on a raw
  pthread *and* a genuine `dispatch_async` GCD worker. (Chez analogue of racket's
  ffi2-callback SIGILL.)
- **But Homebrew Chez is a *threaded* build**, so `Sactivate_thread()` /
  `Sdeactivate_thread()` around the callback body makes a foreign thread safe to
  enter Scheme — both the pthread and the GCD worker survive when activated. This
  is a capability **racket lacks** (its ffi2 callbacks have no such fix).

### 4.2 Decision
**Stay outbound.** Chez → Swift via the `foreign-procedure` C-ABI on opaque
pointers; keep `foreign-callable` callback *creation* in Scheme registered behind
the **native Swift trampolines** (`DelegateBridge`/`BlockBridge`), with GC pinning
(`GCPrevention` / `lock-object`). **Reject** Swift-embeds-Chez-C-API inbound
(larger surface, same foreign-thread problem one level down, no evidence needed).

The chez-specific addition: the native trampolines (and the `dispatch.sls`
`foreign-callable` wrappers) **`Sactivate_thread`-wrap their entry into Scheme**,
so callbacks that legitimately fire on background threads (NSURLSession
completions, `dispatch_async` compute) are safe **without** forcing a main-thread
bounce. The main-thread bounce (`cocoa.sls` `dispatch_async_f`) remains for
**UI-affecting** work — an AppKit requirement, *not* a Chez-safety requirement.
(Recorded: **ADR-0016**, extending ADR-0007.)

This is ADR-0010-aligned: the native library owns the hard concern (threading,
lifetimes); the spike confirms we keep and *deepen* the current architecture.

## 5. Per-concern disposition

**stays Chez (thin/idiomatic)** / **stays native** / **rehomed (de-Common)** /
**deleted**. Note how little *moves* — the spike's whole point.

| surface | disposition | notes |
|---|---|---|
| `emit_class.rs` dispatch (typed `foreign-procedure`) | **stays Chez** | ADR-0015 §2; the typed call is already the native floor |
| `types.sls` ftypes + string/array/dict marshalling | **stays Chez** | ADR-0015 §3; cheap crossings, no native win |
| `objc.sls` guardian + autoreleasepool + NSError | **stays Chez** | ADR-0007; idiomatic Chez lifetime model |
| `cocoa.sls` menu / main-thread bounce / NSView | **stays Chez (thin) over native GCD** | UI main-thread bounce stays |
| `dispatch.sls` block/delegate/dynamic-class `foreign-callable`s | **stays Chez over native trampolines** | §4; **add `Sactivate_thread` wrap** for background callbacks (ADR-0016) |
| `BlockBridge` / `DelegateBridge` / `GCPrevention` (native) | **stays native, deepened** | §4; trampolines gain thread-activation |
| `ffi.sls` dylib loader + `aw_common_*` wrappers | **stays Chez; rename symbols** | dylib already mandatory (ADR-0005); `aw_common_*` → `aw_chez_*` |
| `APIAnywareCommon` 4 files chez uses (ClassLookup, MemoryManagement, AutoreleasePool, StringConversion) | **rehomed → `APIAnywareChez/ChezRuntime.swift`** | ADR-0011; renamed `aw_chez_*` |
| `APIAnywareCommon` (`MessageSend`, `StructMarshal`, `ObservationBridge`, dir) | **deleted** | chez uses none; racket already de-shared on `main` |
| pure-Chez fallbacks for the dylib | **none exist** | chez was built dylib-mandatory from the start (unlike racket) |

**Hermetic isolation (ADR-0011):** racket already de-shared on `main`, so chez is
the last *real* `APIAnywareCommon` consumer. After chez rehomes its 4 files and
drops the dependency, **this grove turns out the lights** — deletes
`APIAnywareCommon`, the inert Gerbil stub, and its `swift/Package.swift` target.

## 6. Consequences and build sequence

- **The emitter does *not* change for dispatch/marshalling** — the surprising,
  load-bearing result. Chez's adoption of ADR-0010 is mostly *recognising* it is
  already compliant, plus the ADR-0011 de-Common and the ADR-0016 thread-safety
  deepening.
- **The dylib stays mandatory** (already true; no fallbacks to delete).
- **Performance:** unchanged on the hot path (it was already optimal); the
  ADR-0016 work *fixes a latent crash* (background callbacks) rather than speeding
  anything up.
- **Build order (execution leaves, root-level siblings):**
  - **020** — de-Common + turn out the lights (merge `main` first; rehome 4 files
    → `ChezRuntime.swift`; rename `aw_common_*` → `aw_chez_*` in `ffi.sls` +
    `runtime/README.md`; drop the `Package.swift` dependency; delete
    `APIAnywareCommon` + Gerbil stub + its target; Swift tests green; regenerate).
  - **030** — background-thread callback safety (ADR-0016): `Sactivate_thread`-wrap
    the native trampolines + `dispatch.sls` `foreign-callable` wrappers; resolve
    the per-thread autoreleasepool vs process-wide guardian interaction; add a
    synthetic background-callback test. *(Gate: if no current sample app exercises
    background callbacks, this may narrow to "wire + synthetic test" or defer via
    inbox — but the decision/ADR stands either way.)*
  - **040** — regenerate the full pipeline for chez + **VM-verify every chez
    sample app** via TestAnyware (standing rule; CLI smoke does not satisfy it).

## 7. Open items (carried to execution leaves)

1. **`Sactivate_thread` bookkeeping** — the `int` disposition; genuinely new
   thread contexts may need `Sdestroy_thread` to avoid leaks across many
   short-lived worker callbacks. (030)
2. **Per-thread autoreleasepool vs process-wide guardian** under an activated
   background callback — the trampoline wrapper must push its own
   `@autoreleasepool` and not assume main-thread guardian-drain timing. (030)
3. **De-Common merge ordering** — this worktree predates racket's de-share on
   `main`; leaf 020 must merge `main` before editing `Package.swift`. (020)
4. **Confirm the Gerbil stub is truly inert** before deleting it. (020)
